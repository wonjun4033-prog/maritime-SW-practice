2) Sequence Diagram

장비 점검 등록(Inspection 등록) 프로세스로 대표 시퀀스를 구성함.

등장 객체 (Lifeline)

Manager
Equipment System (System)
Equipment
Inspection
Alert

시퀀스 메시지 흐름 
① Manager → System
selectEquipment(equipmentId)
장비를 선택하며 점검을 시작함.

② System → Equipment
getEquipmentInfo()
장비 상세 정보 요청.

③ Equipment → System
return equipmentInfo

④ Manager → System
registerInspection(result, date)
점검 내용 입력.

⑤ System → Inspection
createInspectionRecord()
점검 기록 객체 생성.

⑥ Inspection → System
return inspectionId

⑦ System → Equipment
updateStatus(result)

⑧ Equipment → System
return statusUpdated

⑨ System → Alert (조건부)
sendAlertIfNeeded(equipmentId, status)
점검필요/교체필요 상태면 알림 생성.

⑩ Alert → System
alertCreated()

⑪ System → Manager
displayCompletionMessage()

시퀀스 다이어그램 텍스트 형태 요약
Manager → System : selectEquipment()
System → Equipment : getEquipmentInfo()
Equipment → System : equipmentInfo
Manager → System : registerInspection(result)
System → Inspection : createInspectionRecord()
Inspection → System : inspectionId
System → Equipment : updateStatus(result)
Equipment → System : statusUpdated
System → Alert : sendAlertIfNeeded()
Alert → System : alertCreated
System → Manager : displayCompletionMessage()
